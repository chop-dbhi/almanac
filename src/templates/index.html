Components

Everything is a component -- databases, files, columns, code, processes, web services, etc. Components are a model for representing these entities as coarse as a system or as granular as a piece of data.

Relationships

Components rarely stand alone. Relationships are connections between components such as a referential constraint between columns in a relational database or a semantic link between two data elements in independent systems.

Resources

Resources provide structure and organization for related components, generally extracted from the same physical backend.

Collections

Collections are a named set of resources commonly created for a specific project or domain.
